GUÍA PARA REALIZAR LA POBLACIÓN DE DATOS USANDO POSTMAN

Antes de realizar la población de datos, es obligatorio levantar el servidor desde la terminal.

INICIAR EL SERVIDOR

Primero abre la terminal en la carpeta del proyecto backend y ejecuta:

npm run dev


Una vez que el servidor esté corriendo correctamente, recién se podrá realizar la población de datos mediante Postman.

LOGIN COMO ADMIN (PASO OBLIGATORIO)

POST →

http://localhost:3000/api/auth/login


Body JSON:

{
  "correo": "admin_rodolfo@system.com",
  "password": "admin123"
}


Después del login, copiar el TOKEN que devuelve la respuesta.
Este token se debe enviar en todas las peticiones siguientes.

Header obligatorio en Postman:

Authorization: Bearer TU_TOKEN_AQUI

CREAR CATEGORÍA

POST →

http://localhost:3000/api/categorias


Body JSON:

{
  "nombre": "Matemática"
}

CREAR SUBCATEGORÍA

POST →

http://localhost:3000/api/subcategorias


Body JSON:

{
  "nombre": "Álgebra",
  "categoriaId": "ID_DE_LA_CATEGORIA"
}


Nota:
El ID_DE_LA_CATEGORIA se obtiene consultando las categorías creadas:

GET http://localhost:3000/api/categorias

CREAR NIVEL DE DIFICULTAD

POST →

http://localhost:3000/api/niveles


Body JSON:

{
  "nombre": "Fácil"
}


Se pueden crear más niveles como:

Medio

Difícil

CREAR RANGO DE EDAD

POST →

http://localhost:3000/api/rangos


Body JSON:

{
  "descripcion": "5-7 años",
  "edadMin": 5,
  "edadMax": 7
}

VERIFICAR DATOS POBLADOS

Para verificar que los datos se guardaron correctamente, usar los siguientes endpoints:

GET http://localhost:3000/api/categorias
GET http://localhost:3000/api/subcategorias
GET http://localhost:3000/api/niveles
GET http://localhost:3000/api/rangos


Todos deben enviarse con el Header:

Authorization: Bearer TU_TOKEN

ORDEN CORRECTO DE POBLACIÓN

Iniciar servidor (npm run dev)

Login como admin

Crear categorías

Crear subcategorías

Crear niveles

Crear rangos de edad

CONCLUSIÓN

Siguiendo estos pasos en el orden indicado, la base de datos quedará correctamente poblada, con relaciones válidas y respetando el control de acceso por roles implementado en el sistema.
